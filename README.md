# Uptime Checker Service (Сервис мониторинга доступности сайтов)

Высокопроизводительный микросервис на языке **Golang** для параллельного мониторинга доступности веб-сайтов (Uptime) с архитектурой, готовой к высоким нагрузкам. Оформлен в рамках подготовки к магистратуре и техническим собеседованиям.

## Ключевые фичи и технологии
* **Параллельная обработка (Concurrency):** Проверка сайтов выполняется конкурентно с помощью пула горутин (`go routines`) и синхронизируется через группы ожидания (`sync.WaitGroup`).
* **Потоковая запись данных (Stream Logging):** Результаты проверок собираются через потоковые каналы (`channels`) и сохраняются в фоновом режиме (формат JSON Lines).
* **Архитектурный паттерн Clean Architecture:** Код строго разделен на независимые слои: `Domain` (модели), `Repository` (хранение данных), `Service` (бизнес-логика), `Handler` (сетевые интерфейсы).
* **Zero-Allocation HTTP-стриминг:** Реализован собственный алгоритм побайтовой сборки JSON-массива в HTTP-хендлере, исключающий тяжелый парсинг данных в оперативной памяти.
* **12-Factor App Config:** Конфигурация приложения полностью вынесена в переменные окружения операционной системы (порты, интервалы таймеров, списки целевых сайтов).

## Структура проекта
* `cmd/app/main.go` — Инициализация слоев (Dependency Injection), запуск таймеров и HTTP-сервера.
* `internal/config/` — Модуль парсинга конфигурации среды.
* `internal/domain/` — Бизнес-модели данных приложения (`CheckResult`).
* `internal/repository/` — Слой работы с постоянным хранилищем метрик.
* `internal/service/` — Ядро системы, конкурентный обход и логика валидации сетевых ответов.
* `internal/handler/` — HTTP API эндпоинт (`/metrics`) для отдачи собранной статистики.
